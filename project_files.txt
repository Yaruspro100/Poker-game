=== ОПИСАНИЕ ФАЙЛОВ ПРОЕКТА POKER-GAME ===

================================================================================
КОНФИГУРАЦИЯ И НАСТРОЙКИ
================================================================================

  .gitignore
Исключает из Git папку node_modules и файл .env с секретными данными

  server/package.json
Зависимости сервера: Express (веб-сервер), Socket.IO (WebSocket), PostgreSQL (база данных),
JWT (токены), bcrypt (хеширование паролей), CORS, dotenv

================================================================================
КЛИЕНТСКАЯ ЧАСТЬ - АУТЕНТИФИКАЦИЯ (client/js/auth/)
================================================================================

  forms.js
Обработка форм входа и регистрации:
- Валидация полей (логин 3-20 символов, пароль минимум 6)
- Отправка данных на сервер через fetch API
- Сохранение токена и перенаправление в меню
- Обновление UI навигации (показ имени пользователя и баланса фишек или кнопок входа/регистрации)
- Обработка выхода из системы

  session.js
Управление сессией пользователя в localStorage:
- Сохранение и удаление JWT токена
- Сохранение данных пользователя (id, username, chips)
- Получение текущего пользователя из localStorage
- Синхронизация данных с сервером через API /api/profile/me

================================================================================
КЛИЕНТСКАЯ ЧАСТЬ - ИГРОВАЯ ЛОГИКА (client/js/game/)
================================================================================

  actions.js
Управление действиями игрока за столом:
- Обработка кнопок Fold, Check, Call, Raise, All-in
- Отправка действий на сервер через Socket.IO
- Управление слайдером ставок и валидация сумм
- Показ/скрытие доступных действий в зависимости от состояния игры

  cards.js
Отрисовка карт на столе:
- Рендеринг карманных карт игроков (своих и чужих)
- Отображение общих карт (флоп, терн, ривер)
- Анимация раздачи карт
- Показ рубашек карт для чужих карт

  players.js
Отрисовка игроков за столом:
- Позиционирование игроков вокруг стола
- Отображение имени, стека фишек, статуса (активен/фолд/all-in)
- Индикация текущего хода (подсветка активного игрока)
- Показ последнего действия игрока (bet, call, fold и т.д.)
- Отображение дилера, малого и большого блайндов

  pot.js
Управление отображением банка:
- Показ основного банка и сайд-потов
- Обновление суммы банка в реальном времени
- Анимация изменения банка

  state.js
Хранение и обновление состояния игры на клиенте:
- Текущие игроки, их стеки и карты
- Общие карты на столе
- Размер банка и текущая ставка
- Чей сейчас ход
- Синхронизация с сервером через Socket.IO события

  ui.js
Управление оверлеями и модальными окнами:
- Показ результатов раздачи (кто выиграл, какая комбинация)
- Оверлей ожидания начала игры
- Сообщения об ошибках
- Анимации переходов между состояниями

  main-game.js
Главный файл игровой страницы:
- Инициализация Socket.IO подключения
- Подписка на все игровые события (gameState, playerAction, roundEnd и т.д.)
- Координация всех игровых модулей (cards, players, actions, pot)
- Обработка переподключений и ошибок

================================================================================
КЛИЕНТСКАЯ ЧАСТЬ - УТИЛИТЫ (client/js/utils/)
================================================================================

  create-room.js
Форма создания новой покерной комнаты:
- Ввод названия комнаты, пароля (опционально)
- Настройка параметров (малый блайнд, начальный стек, макс. игроков)
- Отправка запроса на создание комнаты
- Перенаправление в созданную комнату

  lobby.js
Лобби со списком доступных комнат:
- Загрузка списка комнат с сервера
- Отображение информации о комнате (название, игроки, блайнды)
- Кнопки подключения к комнате
- Автообновление списка комнат

  logger.js
Лог событий игры:
- Добавление записей в игровой лог (действия игроков, результаты)
- Типы сообщений: обычные, системные, победные
- Автопрокрутка к последнему сообщению
- Ограничение истории до 100 записей

  profile.js
Страница профиля пользователя:
- Загрузка и отображение данных профиля (username, баланс фишек)
- История игр (выигрыши/проигрыши)
- Статистика игрока

СЕРВЕРНАЯ ЧАСТЬ - КОНФИГУРАЦИЯ (server/config/)
================================================================================

  db.js
Подключение к PostgreSQL базе данных:
- Настройка пула соединений через библиотеку pg

================================================================================
СЕРВЕРНАЯ ЧАСТЬ - КОНТРОЛЛЕРЫ (server/controllers/)
================================================================================

  authController.js
Контроллер аутентификации:
- register: регистрация нового пользователя (хеширование пароля bcrypt, проверка уникальности)
- login: вход (проверка пароля, генерация JWT токена, установка cookie)
- logout: выход (удаление cookie с токеном)

  profileController.js
Контроллер профиля:
- getProfile: получение данных текущего пользователя (из JWT токена)
- updateProfile: обновление данных профиля
- getStats: получение статистики игр пользователя

================================================================================
СЕРВЕРНАЯ ЧАСТЬ - MIDDLEWARE (server/middleware/)
================================================================================

  jwtMiddleware.js
Middleware для проверки JWT токенов:
- Извлечение токена из cookie
- Верификация токена и декодирование данных пользователя
- Добавление данных пользователя в req.user
- Обработка ошибок (невалидный/истекший токен)

================================================================================
СЕРВЕРНАЯ ЧАСТЬ - ИГРОВАЯ ЛОГИКА (server/game/)
================================================================================

  roomManager.js
Менеджер покерных комнат:
- Создание и удаление комнат
- Добавление/удаление игроков из комнаты
- Хранение списка активных комнат
- Получение информации о комнате
- Управление состоянием комнаты (ожидание/игра)

  gameLogic.js
Основная игровая логика покера:
- Запуск новой раздачи (сбор блайндов, раздача карт)
- Обработка действий игроков (fold, check, call, raise, all-in)
- Переход между улицами (префлоп → флоп → терн → ривер)
- Определение победителя раздачи
- Распределение банка между победителями
- Управление очередностью ходов

  deck.js
Колода карт:
- Создание колоды из 52 карт (4 масти × 13 рангов)
- Перемешивание колоды (алгоритм Фишера-Йетса)
- Раздача карт игрокам и на стол

  handEvaluator.js
Оценка покерных комбинаций:
- Определение силы руки (роял-флеш, стрит-флеш, каре, фулл-хаус и т.д.)
- Сравнение рук игроков для определения победителя
- Учет кикеров при одинаковых комбинациях
- Возврат названия комбинации на русском языке

  potManager.js
Управление банками:
- Расчет основного банка
- Создание сайд-потов при all-in игроков
- Распределение банков между победителями
- Учет максимальных ставок каждого игрока

  chipWallet.js
Управление балансом фишек игроков:
- Списание фишек при ставках
- Начисление выигрышей
- Проверка достаточности фишек для действия
- Синхронизация баланса с базой данных

  gameState.js
Сохранение и восстановление состояния игры:
- Сериализация состояния комнаты в JSON
- Сохранение в базу данных или файл
- Восстановление игры после перезапуска сервера
- Обработка переподключений игроков

================================================================================
СЕРВЕРНАЯ ЧАСТЬ - МАРШРУТЫ (server/routes/)
================================================================================

  authRoutes.js
Маршруты аутентификации:
- POST /api/auth/register - регистрация
- POST /api/auth/login - вход
- POST /api/auth/logout - выход

  profileRoutes.js
Маршруты профиля:
- GET /api/profile/me - получить данные текущего пользователя (требует JWT)
- PUT /api/profile/me - обновить профиль
- GET /api/profile/stats - получить статистику игр

================================================================================
СЕРВЕРНАЯ ЧАСТЬ - ГЛАВНЫЙ ФАЙЛ
================================================================================

  server.js
Главный файл сервера:
- Инициализация Express приложения
- Настройка middleware (CORS, cookie-parser, body-parser)
- Подключение маршрутов (auth, profile)
- Настройка Socket.IO для WebSocket соединений
- Обработка игровых событий через Socket.IO (joinRoom, playerAction и т.д.)
- Раздача статических файлов из папки client
- Запуск сервера на порту из .env
